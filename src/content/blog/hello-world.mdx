Setiap developer punya cerita tentang baris kode pertama mereka. Saya masih ingat betul bagaimana rasanya melihat teks "Hello, World!" muncul di browser untuk pertama kali — sebuah momen sederhana yang membuka pintu ke dunia yang luar biasa.

## Mengapa 100 Days 100 Web Apps?

Belajar coding tanpa tujuan konkret sangat mudah untuk menyerah di tengah jalan. Saya butuh struktur. Saya butuh *accountability*. Dan lebih dari itu, saya butuh **portofolio nyata** yang bisa membuktikan kemampuan saya ke calon klien.

Maka lahirlah tantangan ini: 100 hari, 100 aplikasi web, dari yang sederhana sampai kompleks.

## Fase 1: Fondasi

Di fase pertama ini, saya fokus pada hal-hal fundamental:

- **Next.js 15** dengan App Router
- **TypeScript** untuk type safety
- **Tailwind CSS v4** untuk styling modern
- **Vercel** untuk deployment

```typescript
// Contoh sederhana: komponen pertama saya
interface GreetingProps {
  name: string;
}

export function Greeting({ name }: GreetingProps) {
  return (
    <h1 className="text-2xl font-bold">
      Hello, {name}! 👋
    </h1>
  );
}
```

## Pelajaran Minggu Pertama

Konsistensi mengalahkan intensitas. Lebih baik coding 2 jam setiap hari daripada marathon 14 jam sekali seminggu. Otak kita membutuhkan waktu untuk mencerna dan mengkonsolidasi pengetahuan baru.
